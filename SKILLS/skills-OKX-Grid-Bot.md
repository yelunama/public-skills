---

## 一、产品概述

OKX Grid Trading Bot 是一款基于 **Claude AI** 驱动的智能网格交易助手，通过 Telegram 提供全中文交互界面。

用户用自然语言描述需求（如「给我推荐一个适合现在行情的网格」），Bot 自动完成：**实时行情分析 → 策略推荐 → 参数生成 → 一键下单**，无需用户具备专业交易知识。

---

## 二、输入输出层

### 2.1 用户输入方式

用户通过 Telegram 消息与 Bot 对话，无需任何配置文件或代码。

| 输入类型 | 示例 | 说明 |
|----------|------|------|
| 自然语言问询 | 「现在适合开网格吗」 | AI 分析当前行情后给出建议 |
| 指定币种分析 | 「帮我分析一下 BTC」 | 拉取实时数据 + Claude 深度分析 |
| 策略推荐 | 「给我推荐一个币做网格」 | 全市场扫描后推荐最优标的 |
| 下单指令 | 「买 0.01 个 BTC」 | 展示订单摘要，确认后执行 |
| 持仓查询 | 「我的机器人跑得怎么样」 | 检查所有网格机器人状态 |
| 会话重置 | `/clear` | 清空对话历史，重置上下文 |

### 2.2 系统输出内容

| 输出类型 | 内容 | 触发场景 |
|----------|------|----------|
| 行情分析报告 | 价格、ATR、EMA 趋势、资金费率、量比 | 用户询问行情/策略 |
| 策略推荐 | 推荐币种、网格区间、格数、投入金额建议 | 用户请求推荐 |
| 订单摘要 | 币种/方向/数量/估算金额，等待用户确认 | 用户发起下单 |
| 执行回执 | 订单 ID、成交状态 | 下单成功后 |
| 持仓报告 | 机器人运行状态、盈亏、当前价格 | 用户查询持仓 |
| 错误提示 | 余额不足、币种不存在等 | 执行异常时 |

## 三、数据层：实时行情预取

### 3.1 数据架构思路

为降低 AI 响应延迟，系统采用 **Python 预取 + 数据注入** 架构：

```
用户发消息
  │
  ├─ Python 并行拉取 OKX 实时数据（3-7s）
  │    ├─ Ticker（当前价、24h 涨跌）
  │    ├─ ATR 4H（波动率）
  │    ├─ EMA20 / EMA60（趋势判断）
  │    ├─ 量比（近期成交量相对均值）
  │    └─ 资金费率（永续合约）
  │
  └─ 将数据注入 Claude 上下文 → Claude 直接推理，无需再调数据接口
```

### 3.2 预取模式

| 模式 | 触发条件 | 数据范围 | 耗时 |
|------|----------|----------|------|
| 全市场快照 | 未指定币种（如「推荐一个」） | Top 15 标的，ATR + 资金费率 | ~6-8s |
| 单币预取 | 消息中包含明确币种（如「BTC」） | 单币完整指标 | ~2-3s |
| 无预取 | 交易/下单/持仓类问题 | — | <1s |

### 3.3 支持的数据指标

| 指标 | 说明 | 用途 |
|------|------|------|
| 当前价 / 24h 涨跌 | OKX Ticker | 趋势判断 |
| ATR%（4H 真实波动率） | 近 20 根 4H K 线 | 网格区间宽度建议 |
| EMA20 / EMA60 | 均线系统 | 多空趋势判断 |
| 量比 | 近 4 根 / 前 16 根均量 | 识别放量异动 |
| 资金费率 | 永续合约每 8h 费率 | 合约做空/做多成本评估 |

## 四、决策层：Claude AI 分析引擎

### 4.1 核心能力

Claude AI 基于注入的实时数据，自主完成以下推理：

- **行情判断**：识别当前是震荡行情、趋势行情还是高波动行情，给出网格适用性评分
- **参数生成**：根据 ATR、EMA 自动计算网格上下轨、格数、单格利润预估
- **风险提示**：标注资金费率异常、波动率过高等风险信号
- **标的筛选**：从 Top 15 标的中综合评分，推荐最优网格标的

### 4.2 分析输出要求

AI 的每次分析必须包含：

| 输出项 | 说明 |
|--------|------|
| 推荐理由 | 说明为什么推荐该币/策略，数据如何支持判断 |
| 具体参数 | 价格区间、格数、投入金额建议、止盈止损 |
| 风险提示 | 当前行情的潜在风险，不回避不美化 |

> AI 不输出模板化内容，每次分析结合当前实时数据进行真实推理。

### 4.3 工具调用能力（Tool Use）

Claude 在需要时可主动调用以下工具补充分析：

| 工具 | 用途 |
|------|------|
| `market_get_ticker` | 获取指定币种实时价格 |
| `market_get_candles` | 获取 K 线数据 |
| `market_get_funding_rate` | 获取资金费率 |
| `market_check_instrument` | 验证币种是否在 OKX 上市 |
| `account_get_balance` | 下单前查询账户余额 |
| `trade_place_order` | 执行现货 / 合约买卖 |
| `grid_get_orders` | 查询网格机器人运行状态 |

## 五、交易执行层

### 5.1 下单流程

```
用户说「买 / 卖 / 做多 / 做空 / 下单」
  │
  ├─ Claude 检查账户余额（account_get_balance）
  ├─ 展示订单摘要给用户确认：
  │    币种 / 方向 / 数量 / 估算金额 / 当前价
  │
  └─ 用户回复「确认」后 → 执行 trade_place_order
       └─ 返回订单 ID + 成交状态
```

### 5.2 支持的订单类型

| 类型 | 说明 |
|------|------|
| 现货买入 / 卖出 | 市价单 / 限价单 |
| 永续合约做多 / 做空 | 自动识别 `-SWAP` 合约 |
| 网格机器人创建 | 指定区间、格数、投入金额 |
| 网格机器人停止 | 停止并可选是否平仓 |

### 5.3 安全机制

- **二次确认**：所有下单操作必须用户回复「确认」后才执行
- **余额检查**：下单前自动验证余额是否充足
- **币种验证**：自动识别用户说的中文名/英文代号是否在 OKX 有对应交易对
- **非加密资产拦截**：识别黄金（XAU）、外汇等非加密资产，直接告知不支持

## 六、会话管理

### 6.1 对话历史

| 配置项 | 值 | 说明 |
|--------|----|------|
| 最大历史轮数 | 16 轮 | 超出后自动丢弃最早的消息 |
| 单条消息字符上限 | 3,000 字 | 超出自动截断，防止 Token 浪费 |

### 6.2 多用户隔离

每个 Telegram 用户（user_id）拥有独立的对话历史，互不干扰。

### 6.3 会话重置

用户发送 `/clear` 命令，清空当前对话历史，适用于：
- 上下文混乱时重置
- 切换分析方向时清空旧数据

## 七、系统架构

### 7.1 组件总览

```
Telegram Bot（bot.py）
  │
  ├─ claude_handler.py     # AI 对话主逻辑，数据注入 + Claude 工具循环
  ├─ okx.py                # OKX REST API 封装（行情 + 账户 + 交易）
  ├─ tools.py              # Claude Tool Use 定义与执行器
  └─ config.py             # API Key / Model 配置
```

### 7.2 关键设计原则

| 原则 | 说明 |
|------|------|
| Python 只取数据，Claude 才做分析 | 所有推理、报告、推荐均由 Claude 生成，不用模板 |
| 并行预取 | ThreadPoolExecutor 并行拉多路数据，最小化等待时间 |
| 无超时限制 | Claude 跑到完成为止，保证输出质量 |
| 工具并行执行 | 多个 Tool Use 并发调用，缩短多工具场景耗时 |

## 八、术语表

| 术语 | 定义 |
|------|------|
| ATR（Average True Range） | 真实波动率，用于衡量价格波动幅度，网格区间宽度的核心参考指标 |
| EMA（指数移动平均线） | EMA20 / EMA60 用于判断短中期价格趋势 |
| 量比 | 近 4 根 K 线成交量 / 前 16 根均值，>1.5 为放量，<0.5 为缩量 |
| 资金费率 | 永续合约多空双方每 8h 互相支付的费率，正值偏多头拥挤，负值偏空头拥挤 |
| Tool Use | Claude 主动调用外部工具（行情/下单/查仓）的能力 |
| 数据预取 | Python 在 Claude 处理前并行拉取市场数据并注入上下文，减少 AI 调用工具的次数 |
| 槽位（Slot） | 预留占位符，本版本无硬性槽位限制 |
| VectorEngine | Claude API 的代理端点，`https://api.vectorengine.ai/` |

# Contributing

## English

### Rules

1. Put each skill in `skills/<skill-name>/`.
2. Use lowercase letters, digits, and hyphens for skill folder names.
3. Include a `SKILL.md` in every skill folder.
4. Keep each pull request focused on one skill or one repository-level change.
5. Update documentation when a contribution changes repository usage or structure.

### Pull Request Checklist

- The skill folder name follows hyphen-case.
- `SKILL.md` explains what the skill does and when to use it.
- Optional resources are added only when they are actually needed.
- Repository links and references are correct.

### Review Model

- Contributors submit changes through pull requests.
- Maintainers review before merging into `main`.

## 中文

### 规则

1. 每个 skill 必须放在 `skills/<skill-name>/` 目录下。
2. skill 目录名只能使用小写字母、数字和连字符。
3. 每个 skill 目录必须包含 `SKILL.md`。
4. 每个 Pull Request 应聚焦单个 skill 或单个仓库级改动。
5. 当贡献影响仓库用法或结构时，必须同步更新文档。

### Pull Request 检查项

- skill 目录名符合 hyphen-case 规范。
- `SKILL.md` 清楚说明 skill 的用途和触发场景。
- 可选资源目录仅在确有需要时添加。
- 仓库链接和引用信息正确。

### 审核方式

- 所有贡献通过 Pull Request 提交。
- 合并到 `main` 之前必须经过维护者审核。
